操作系统安全技术

工业论坛 I9

一、论坛介绍

操作系统是计算机的核心基础软件，负责管理底层硬件设备和支撑上层应用服务。鉴于操作系统的重要地位，近年来，CrowdStrike软件错误更新导致微软系统崩溃、WannaCry病毒导致微软系统数据加密、DirtyCow漏洞导致Linux系统恶意提权等操作系统安全事件频发，造成了严重的经济损失和社会影响；根据公开的CVE漏洞数目排名中，操作系统在所有程序类型排名中长期位居前列。因此，通过技术手段保障操作系统安全是工业界和学术界关注的重要问题。

操作系统安全技术论坛聚焦操作系统安全技术的学术研究前沿和工业实践经验，具体技术内容包括操作系统的安全测试、缺陷检测、漏洞挖掘、安全防护、供应链分析等，尤其关注新型技术方法在开源/国产操作系统的实际应用，为各领域专家协同创新、合作共享提供交流平台，助力构建更完善的操作系统安全技术体系，推动开源/国产操作系统的高质量健康发展。欢迎从事操作系统、网络安全、软件工程等专业的专家和学者前来参加。

四、论坛报告嘉宾简介

1. 报告人：游伟 副教授

题目：Linux内核漏洞利用代码自动化生成技术研究

摘要：内核漏洞给计算机系统的安全性带来了严峻挑战。许多安全事件的根源来自于内核漏洞。目前已有大量研究工作关注内核漏洞的检测、触发和修复，而内核漏洞的自动化利用尚未得到足够多的重视。此次报告将介绍Linux内核漏洞的利用代码端到端自动化生成技术研究，对于内核堆内存越界写漏洞的触发代码，自动化分析内核中的能力跳板对象，进行能力串联，最终达到提权的攻击效果。相关技术可直接应用于实际的安全攻防对抗。

简介：游伟，中国人民大学副教授，博士生导师，计算机系副系主任，入选国家级青年人才项目。长期从事软件漏洞自动化挖掘利用，以及二进制程序的动态和静态分析。游伟博士在信息安全和软件工程领域国际顶级学术会议/期刊论文上发表论文二十余篇，获得最佳论文奖一次，最佳应用安全论文提名奖两次，获得CCF科学技术奖自然科学一等奖。

2. 报告人：李勇钢 副教授

题目：面向不可信设备的驱动运行时安全防护方法研究

摘要：2025年爆出的某知名GPU“后门事件”另我们不得不思考不可信设备带来的底层安全问题。硬件驱动是外部设备与操作系统建立逻辑关系的关键组件，其折射出的安全问题对于防护不可信设备具有重要意义。但是，许多设备驱动是闭源的，致使其内部执行逻辑难以被精确地刻画出来，进而导致所有基于源码的安全审计方法失去效果。本研究采用软硬件协同的方法，对闭源驱动的运行时行为进行追踪和检测，从而发现潜在的危险动作。从当前已取得的成效来看，该方法能够摆脱对源码的依赖性问题，并利用简单的安全策略取得良好的防御效果。

简介：李勇钢，2019年博士毕业于中国科学技术大学，之后在香港中文大学（深圳）从事博士后研究，现在在中国矿业大学计算机学院任职副教授，担任网络空间安全系主任，主要研究领域是系统安全和虚拟化，在国内外知名刊物与会议中发表论文40余篇，包括ACM CCS、IEEE TC、IEEE TDSC、IEEE TSC、ACSAC和计算机学报等，并获得安全旗舰会议CCS“杰出论文奖”。目前，李勇钢博士主要致力于基于软硬件协同的未知软件运行时安全检测与保护研究。

3. 报告人：李强 安全专家

题目：AI Agent runtime安全：现状、挑战以及解法

摘要：AI Agent的广泛应用带来了新的安全问题，本议题会详细介绍AI Agent运行环境的安全风险以及我们的安全事件，主要包括对AI Agent以及AI Agent的整体风险做概括性描述、传统的系统安全在AI Agent的面临的挑战以及AI Agent runtime的具体风险、业界目前针对AI Agent runtime的安全方案以及蚂蚁集团内部的实践。

简介：李强，蚂蚁集团安全专家，专注于系统安全，目前在蚂蚁负责基础设施安全，涉及领域包括Linux内核/虚拟化/容器/云原生安全，曾在国内外多个安全会议上发表研究议题。

4. 报告人：贾相堃 副研究员

题目：模糊测试驱动生成：从自动化到智能化

摘要：编写模糊测试驱动进行测试是函数库测试的最有效方案之一，如何编写有效驱动是亟需解决的问题。团队提出了自动化方案AFGen，重点解决了驱动自动生成中上下文依赖、参数初始化和触发路径约束等难题，能力优于Google FUDGE等同类方案。进一步，团队提出了智能化方案PromeFuzz，通过引入大模型并进行系统性的知识增强，解决了生成能力弱、误报率高等问题，能力优于Google OSS-Fuzz-Gen等AI方案。相关成果形成论文发表于IEEE S&P 2024、ACM CCS 2025，并应用于OpenHarmony系统基础函数库测试。

简介：贾相堃，中国科学院软件研究所副研究员，CCF高级会员/计算机安全专委会执行委员，中国网络空间安全学会软件与系统安全专委会成员，中国科学院青促会成员，获中国科学院软件所“所优青”项目支持。主要研究方向为软件与系统安全、漏洞挖掘、分析与利用。主持了国家自然科学基金面上项目、青年基金项目等，参与了国家自然科学基金重点项目、国家重点研发计划等，研究成果应用于华为、蚂蚁等企业。相关研究成果发表于IEEE S&P、ACM CCS、USENIX Security、NDSS、ASE等网络安全和软件工程顶会，多次担任网络安全顶会PC，担任TIFS等顶刊审稿人和信息安全学报等期刊青年编委。

5. 报告人：刘维杰 副教授

题目：Linux原生容器安全隔离增强技术

摘要：本报告将深入探讨Linux原生容器技术的发展路径和安全隔离性演进。从原生容器技术（如runc）的基础原理出发，分析其基于Linux namespace和cgroup实现资源隔离的方式，同时指出容器逃逸和资源滥用等安全问题。为应对这些挑战，报告提出了一系列创新解决方案，包括提升现有MNT ns的隔离机制，增强容器文件系统隔离；引入气球cgroup管理残留资源，以实现namespace和cgroup的高效协同。同时，报告还将探讨基于microVM和LibraryOS的安全容器技术，展望容器技术的未来发展方向，强调容器技术的多元化融合趋势；并以Apple的新型容器框架为例，展示下一代容器技术在文件系统优化、快速启动和跨架构支持等方面的优势。

简介：刘维杰，南开大学副教授，CCF计算机安全专委会执行委员，星绽Asterinas开源社区技术指导委员会常务委员。主要研究方向为系统安全、机密计算，相关研究成果发表于IEEE S&P、CCS、NDSS、ASPLOS、EuroSys、FAST、DSN、ESORICS安全与系统领域顶级会议，授权专利多项，具有丰富的机密计算实践经验。担任国际安全基因计算竞赛iDash出题人和裁判组负责人，大学生计算机系统能力大赛、隐私计算大赛的出题人和评委。

6. 报告人：郑耿 技术专家

题目：从响应到预见：龙蜥操作系统供应链安全的智能化演进之路

摘要：随着软件供应链日益复杂和攻击面持续扩展，传统“发现—修补”式的操作系统漏洞治理模式已难以应对新型威胁。本报告系统梳理操作系统漏洞治理的演进路径，重点探讨两大前沿方向：一是构建覆盖开发、分发、部署全链条的供应链安全治理体系，强化对研发全流程的风险管控；二是融合人工智能技术，实现漏洞的自动化识别、优先级评估等能力。通过案例分析与技术展望，展示如何迈向主动、协同、智能的新一代操作系统安全治理范式。

简介：郑耿，清华大学计算机系毕业，阿里云操作系统实验室技术专家。深耕基础设施和系统安全领域多年，具备扎实的系统软件研发和架构设计经验。现任阿里云基础软件部及龙蜥开源社区的软件供应链安全总架构师，主导软件供应链安全体系的整体设计与落地建设，致力于构建安全、可信、自主可控的基础软件生态。

7. 报告人：马发俊 教授级高工

题目：openKylin社区安全体系建设实践

摘要：开源社区已成为推动协同创新的重要平台载体。安全问题也逐渐凸显并影响开源社区发展。本报告主要聚焦openKlyin社区的安全体系建设探索与实践，首先分析开源社区建设和发展面临的安全挑战，深入讨论项目生命周期各个阶段的安全策略和实践，介绍openKyin开源社区安全体系建设的具体项目案例，剖析通过这些项目提升开源项目安全性的可行性。并探讨参与openKylin开源社区安全体系建设的路径，共同推进开源社区安全体系的持续优化。

简介：马发俊，教授级高工，openKylin开源社区生态合作负责人；国产多语种操作系统技术联盟秘书长，全国信标委多语言国家标准工作组成员，上海市软件行业协会副会长，中国中文信息学会民文专委会常务委员，CCF开源发展委员会委员；长期从事国产基础软件研发和推广工作，多次参与和主持工信部电子发展基金项目、核高基重大专项、上海市软件和集成电路专项项目，获上海市科技进步二等奖和上海市徐汇区拔尖人才等荣誉。取得数项基础软件领域发明专利授权，发表相关学术论文多篇。
