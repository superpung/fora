# Fora

**简洁、快速的会议日程浏览器 —— 浏览每一场议程，标记你在意的内容，让日程在所有设备间同步。**

[English](README.md) · 中文

</div>

Fora 把密集的学术会议日程变得真正好用。一个站点承载多个会议:从首页选择一个
会议,浏览它的主旨报告与各个分论坛,通过给感兴趣的报告加星来构建你的个人日程,
并让它在你使用的每一台设备上同步 —— 无需安装任何应用。

## 功能

- **多会议首页** —— 一个站点,多个会议。选择其一,直接进入完整日程。
- **完整的会议内容** —— 主旨报告、并行分论坛、组织委员会、主办与赞助方,全部
  中英双语呈现。
- **时间线** —— 把所有并行分论坛按时间并排排布,会议当天还会显示一条实时的
  "当前时间"红线。
- **个人日程** —— 给任意报告、分论坛或讲者加星,你的选择会汇成一份可筛选、可
  随时回看的专属日程。
- **跨设备同步** —— 用 GitHub 登录,即可把日程备份到你自己名下的私有 Gist 并
  在各设备间同步。仅浏览无需登录。
- **导出与导入** —— 把日程导出为日历(`.ics`)、表格(`.csv`)、Markdown
  (`.md`),或可再次导入的备份(`.json`)。
- **讲者目录** —— 按姓名、单位或报告搜索讲者,并可按首字母快速定位。
- **分享海报** —— 为整个分论坛或单场报告生成一张简洁的海报图片。
- **浅色与深色** —— 跟随系统,也可手动切换。无任何追踪。

## 你的日程,随处可用

加星是即时且本地保存的 —— 在当前设备上构建日程无需登录。当你希望手机和电脑都
能看到时,用 GitHub 登录即可:Fora 会把你的关注保存在你自己账号下的一个私有
Gist 中,并在各设备之间对账同步。除此之外没有任何数据离开你的浏览器,你也可以
随时在账户菜单里导出或清空数据。

## 开发

Fora 的前端是位于 [`web/`](web/) 的 React + Vite 单页应用;它渲染的数据集由
[`source/`](source/) 下的 Python 流水线离线构建。

```bash
cd web
pnpm install
pnpm dev        # 启动开发服务器
```

架构、自研的数据 schema、构建流水线,以及**如何新增一个会议**,详见
**[AGENTS.md](AGENTS.md)**。

## 数据与致谢

所有会议内容 —— 报告标题、讲者、单位、摘要 —— 版权归各自的主办方所有(CCF 及其
各会议委员会)。Fora 只是把公开的日程信息重新整理成一致、易读的视图。跨设备同步
由 [`@repus/gist-sync`](https://www.npmjs.com/package/@repus/gist-sync) 提供支持。
