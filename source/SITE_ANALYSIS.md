# CCF Chip 2026 官网结构分析（数据源逆向）

网址：https://ccf.org.cn/ccfchip2026 ；会议短名 `ccfchip2026`。
本文档记录该站点的数据来源与议程信息的组织方式，作为本项目解析器的输入依据。

## 1. 站点技术形态
- Vue SPA + 后端 CMS 渲染，页面初始 HTML **不含**议程内容，全部走 jsonp 接口。
- 所有接口需带 `Referer: https://ccf.org.cn/ccfchip2026`。

## 2. 关键接口（见 `fetch_all.py`）
| 步骤 | 接口 | 参数 | 返回 |
|---|---|---|---|
| 1 | `/api/getMeetingIdByMeetingShort.action` | `meetingShort=ccfchip2026` | meetingId = `m1478733396288081920177259911521` |
| 2 | `/api/show.action` | `code=api_channel&meetingId=<id>` | 栏目树（78 个栏目，3 级） |
| 3 | `/api/newsAll.action` | `channelId=<id>` | 该栏目文章数组（含富文本/结构化字段） |
| 资源 | `/cmsFileManager/UeditorImg/*.png\|jpg` ；PDF/附件走 `/web/formDes/download.action?filePath=...` |

原始数据已全部落盘：`raw/api/channels/<channelId>.json`（每个栏目）、`raw/images/`（76 张图）、`raw/files/`（PDF/docx）、`raw/api/manifest.json`（栏目清单+资源映射）。

## 3. 栏目树 → 信息类型
栏目 78 个，按内容形态分三类：

### A. 议程总览（`大会议程` 栏目, type=4）
- 1 篇文章，正文含 **6 张 A4 竖版 PNG**（2481×3508）+ 附件「大会议程一览.pdf」。
- 6 张图内容：
  1. 封面/总览：会议名(中英)、日期 2026/7/17–7/20、地点、联系邮箱、主/承/协办、主席团、**4天×(上午/下午/晚上) 概览矩阵**。
  2. 7/17 签到；7/18 上午 主旨报告（太湖厅AB，逐场：时间+讲者+头衔+题目）；下午 技术论坛时段。
  3. 7/18 下午 18 个并行论坛表（会议室↔CFxx↔论坛名）；「知存之夜」晚宴；7/19 起始。
  4. 7/19 上午 主旨报告；下午 技术论坛论坛表（前 8）。
  5. 7/19 下午论坛表（后 8）；CCF 专委工作会议（3 场，各自时段）；7/20 上午 技术论坛。
  6. 7/20 上午论坛表（14 个）；官网二维码。
- **总览只给到"论坛"层**（CFxx + 会议室 + 论坛名 + 专场赞助商），论坛内部报告不在此。

### B. 技术论坛详情（`技术论坛` 下 48 个子栏目 CF01–CF48, type=4）
- 每个论坛 = 1 篇文章，正文 **无文字，仅 1 张超长竖图海报**（1600 × 26000–37000 px, JPG 5–7MB）。
- 海报内部结构（已逆向确认，见 CF01）：
  ```
  论坛名称
  └ 论坛简介（段落）
  └ 论坛主席（1..n 人）：照片 + 姓名 + 单位/职称 + 个人简介
  └ 论坛讲者：
       对每个报告重复：
         [箭头蓝底横幅 = 报告题目]      ← "（待确认）" 表示 TBD
         演讲人：照片 + 姓名 + 单位/职称
         演讲摘要（可为「待确认」）
         个人简介
  └ 关注二维码
  ```
- **重要**：论坛内各报告**没有单独时间戳**（时间只到总览的时段级 13:30–17:00）。报告题目与摘要可能缺失/待定；源海报偶有题目与摘要不一致（如 CF01 韩军），需按"如实记录+打标记"处理。

### C. 组织/人员/赞助/说明类（结构化字段，无需读图）
- 组织机构（`组织机构` 下 13 个角色栏目）：字段 `NAME_INFO`（姓名）、`MAIN_INFO`（单位+职称）、`FILE_`（照片）。共 **51 人**：大会指导委员会15、大会主席3、大会共同主席1、大会执行主席4、组委会主席5、组委会副主席2、程序委员会主席4、论坛组织主席5、赞助主席3、宣传主席5、网站主席1、出版主席2、注册主席1。
- 合作单位（`合作单位` 下 5 档）：金钻/钻石/铂金/黄金/白银，logo + 名称。
- 文本类：会议介绍（`INTRODUCTION` 字段，注意不是 MAIN_BODY）、程序委员会、论文征集、邀请函、住宿预订、诚邀赞助、资料下载。

## 4. 对数据模型的启示（为什么要自建 schema）
1. **4 层结构**：会议 → 天 → 时段块(主旨报告/技术论坛/晚宴/签到/专委会) → 论坛(CFxx) → 报告。
2. **论坛是一等实体**：有编号、名称、专属会议室、专场赞助商、主席、内嵌报告；不能退化成一个标签。
3. **报告无独立时间**：只有时段级时间窗；schema 必须允许 talk.start 缺省。
4. **大量非报告事项**：签到、茶歇、晚宴、专委工作会议。
5. **多场地**：无锡国际会议中心 + 大会推荐酒店。
6. **中英双语 + 待定(TBD) 状态** 普遍存在。
7. **会议级人物/组织**：主席团、13 类委员会、5 档赞助单位。
