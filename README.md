# conf-scheduler

Parse a conference agenda into a **custom, general-purpose format**, then use it
to drive a web viewer. First sample conference: the 3rd CCF Chip Conference
(第三届 CCF 芯片大会 / CCF Chip 2026), https://ccf.org.cn/ccfchip2026

## Design principles
- **Compatible with / mapped to no existing format** (pretalx / frab / pentabarf
  are all out of scope).
- The schema is designed purely from the real conference data — see
  [`schema/schema.json`](schema/schema.json).
- Core structure: `conference → day → time block (keynotes / forums / banquet /
  registration / committee meetings) → forum (CFxx) → talk`.
  - **A forum is a first-class entity** (code / name / room / session sponsor /
    chairs / embedded talks).
  - **Talks inside a forum have no independent time** — they inherit the block's
    time window; bilingual and TBD states are supported throughout.

## Layout
```
schema/schema.json          Self-designed JSON Schema (draft 2020-12)
data/
  ccfchip2026.json          ★ Full dataset (conforms to the schema, validated)
  forums_detail/CFxx.json   Visually parsed forum details (CF01–CF48; only CF10
                            lacks a source poster)
source/
  SITE_ANALYSIS.md          Reverse-engineering of the site (data sources / APIs)
  fetch_all.py              Crawl all 78 channels + 76 assets -> raw/
  extract_structured.py     Extract structured content (committees / sponsors /
                            text / forum image map) -> extracted/
  slice_poster.py           Slice a very tall forum poster for chunked reading
  batch_slice.py            Batch-slice multiple forum posters
  build_dataset.py          Assemble data/ccfchip2026.json (also writes
                            web/src/data/conference.json)
  validate.py               Validate the data against the schema
  raw/                      Raw dumps: api/channels/*.json, images/, files/,
                            manifest.json
  extracted/                Structured intermediate artifacts
web/
  src/data/conference.json  ← Generated by build_dataset.py; the web app's single
                            data source (do not hand-edit)
```

## Reproduce
```bash
cd source
python3 fetch_all.py          # crawl (done; artifacts in raw/)
python3 extract_structured.py # extract structured content
python3 build_dataset.py      # write data/ccfchip2026.json + web/src/data/conference.json
python3 validate.py           # validate (needs: pip install jsonschema)
```
`build_dataset.py` writes the dataset to both `data/` and `web/src/data/`, so the
two copies stay identical with no manual sync. Output is **deterministic**: with
unchanged source data a rerun is byte-for-byte identical (`extraction.content_sha256`
is a content fingerprint; a real `generated_at` timestamp is written only when
`SOURCE_DATE_EPOCH` is set).
