大语言模型赋能软件研发的效能度量和改进

工业论坛 I4

一、论坛介绍

近年来，大语言模型（LLM）正在不断重塑软件工程的研发范式。一方面，大语言模型赋能软件工程（LLM4SE）的浪潮方兴未艾；另一方面，LLM4SE的实际成效未明。在这样的背景之下，如何科学度量并提升研发效能，已成为学术界与工业界共同关注的焦点话题。实践中，LLM4SE的研发效能度量面临着多重挑战，例如，工具链碎片化、传统度量指标失效、局部效率提升难以转化为全局价值等。为此，本论坛邀请业界一线专家，围绕下列三个话题展开深度研讨：

一、解构LLM对软件工程全生命周期的真实赋能状况。话题内容覆盖从编码辅助到需求建模、架构决策、测试生成与智能运维等各个环节，试图揭示LLM在不同阶段的技术渗透现状与落地挑战。

二、探讨LLM4SE时代研发效能的科学度量体系构建以及标准化工作的意义和价值。

三、展望效能改进的前沿。探讨将LLM4SE技术演进与相应的效能度量进行有机融合的方法和实践。

诚邀您参加本论坛，共同推动LLM4SE背景之下软件研发的质效跃迁。

四、论坛报告嘉宾简介

1. 报告人：肖然 Thoughtworks中国区总经理

题目：软件3.0时代的研发效能治理挑战

摘要：软件3.0时代以意图驱动、大模型为核心，对研发效能治理提出全新挑战。传统DevOps围绕确定性代码构建CI/CD体系，追求高效、可靠的交付能力；而软件3.0的核心逻辑由大模型动态生成，具有概率性特征，使得研发过程从“确定性交付”转向“不确定性治理”。因此，研发效能治理必须从“交付确定性”转向“驾驭不确定性”。新挑战在于构建一个融合“实验-评估-调整”（E²A）的动态循环。我们需要建立新的质量度量标准，持续监控AI的行为风险（如幻觉、偏见），并高效管理提示词、知识库等新资产。最终，需演进为能同时治理稳态系统、运营敏态服务、探索探态AI应用的“三态共治”新体系。

简介：肖然，Thoughtworks数字化专家、中关村智联联盟秘书长，长期深耕企业数字化与智能化转型，专注于AI、敏捷开发、DevOps及数据驱动创新。作为金融科技与数字化转型专家，他担任《财新》《数字银行》专栏作者，并为中行、招行、华为等企业提供战略咨询。近年来，他聚焦“软件3.0时代”下AI技术的行业落地，在TiD、InfoQ等顶级技术峰会发表主题演讲。同时，他积极推动技术社区生态建设，联合发起中国敏捷教练企业联盟（CAC）和DDD China，促进前沿技术在国内的普惠发展。作为上海交通大学高金学院讲师及工商银行研修院特邀专家，他致力于AI人才培养与行业赋能，助力企业探索高价值AI场景。

2. 报告人：向邦宇 阿里巴巴代码平台负责人

题目：Vibe Coding 工具在企业内部的运用和挑战

摘要：AI 发展过程中诞生了许多优秀的 Coding 产品，但非专业开发者需要掌握一些简单的研发知识才能完成研发任务，而这些工具和研发知识的匮乏，都在不同程度上影响非专业开发者的热情。本次演讲将探讨如何构建下一代 Vibe Coding 工具。我将从当前挑战出发，提出以用户为中心、强化工具质量、深化场景适配、支持协作与包容不确定性的核心设计原则与实践，希望给听众带来一些启发和思考。

简介：阿里巴巴代码平台负责人，在代码管理、代码结构化数据处理、代码搜索、代码评审以及编辑器技术等领域拥有丰富的专业知识和实践经验。在阿里，负责了包括 CloudIDE、代码搜索、CodeReview 等多个关键产品的开发与管理，成功引领了代码智能平台的建设与发展。他主导实现的阿里内部多个 AI Coding 工具，包括 Aone Copilot 和 Aone Agent 等，在阿里内部被广泛使用，还主导开发了 AI Development 产品“搭叩”。

3. 报告人：范贵生 副研究员

题目：LLM驱动研发效能提升——企业级代码重复度治理实践与展望

摘要：在大语言模型赋能软件工程的浪潮中，如何科学度量并提升研发效能，已成为业界核心议题。AI技术凭借其处理海量数据与识别深层模式的能力，为这一领域带来了全新的解题思路。在诸多研发效能痛点中，“代码重复度高”是企业普遍面临的挑战，它直接导致维护成本上升、缺陷率增加，并拖慢开发迭代速度。因此，我们选择以“代码重复度治理”为具体抓手，来系统探索LLM技术在企业效能治理中的落地实践。本次分享首先回顾代码重复度这一指标的价值与挑战，然后重点阐述治理方案如何从传统启发式方法，演进至基于LLM的新度量体系，并分享实际落地中的关键经验与效果评估。更进一步，我们的视野从当前实践延伸至未来展望：LLM对效能度量本身将产生双向影响。一方面，它的普及可能使部分传统指标失效；另一方面，它也能赋能构建“效能度量智能体”，实现从数据分析到风险预判的全流程自动化，为研发效能的持续进化开辟新的可能。

简介：华东理工大学副研究员，博导，长期从事智能软件和可信数据空间研究。相关研究成果发表于TSE、TDSE、JSS、TSMC-S、JCST、计算机学报、软件学报等期刊和会议，授权专利多项，获上海市科技进步一等奖1项、上海市科技进步二等奖2项。

5. 报告人：彭丽娟 北京兴云数科高效研发平台的研发效能负责人

题目：以量化技术为锚点，AI赋能研发效能跃迁

摘要：软件3.0时代，基于LLM底座，快速生成研发过程产出物。软件研发过程不仅是以价值交付为中心，更以“大模型+数据”为中心，提供个性化服务。 面对众多研发数据，如何利用现有数据，科学度量提升研发效能是当下热议的话题。在诸多研发效能痛点中，从哪里下手，是从时间占比维度看过程提效点，还有没有其他方法可以事半功倍。本文从量化技术角度，运用蒙特卡罗模拟预测、数据建模等技术找到研发改进的锚点，再基于LLM底座，运用现有知识库，使用AI智能体，进行研发改进、效能提升。 同时展望，在LLM4SE（大语言模型赋能软件工程）浪潮下，对新的研发模式、新的组织形态、新的管理变化、新的度量体系和度量指标等进行探索。

简介：北京兴云数科高效研发平台的研发效能负责人，先后在产品生命周期管理PLM平台、DevOps平台、产品安全等领域负责过多个产品，期间一直从事研发数据度量分析、研发效能提升等工作。致力于LLM4SE环境下，质量赋能产品与高效的智能化软件产品研发工作。
